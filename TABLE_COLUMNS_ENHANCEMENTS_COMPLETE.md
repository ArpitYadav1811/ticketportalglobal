## 🎨 Design System Summary

### Color Palette (All Solid)
- **Blue**: Primary actions, ticket IDs, open status
- **Purple**: Requirements, project actions, returned status
- **Amber**: Support type, on-hold status
- **Green**: Resolved status
- **Red**: Deleted status
- **Slate**: Closed status, text, borders
- **Gradients**: Only for user avatars (purple-pink, blue-cyan)

### Typography
- **Bold**: Ticket IDs, status badges, action labels
- **Semibold**: Names, titles, dates
- **Medium**: Secondary text, groups
- **Sizes**: xs (badges), sm (content), lg (empty state)

### Spacing
- **Padding**: 
  - Badges: px-3 py-1.5 to py-2
  - Buttons: p-2
  - Cells: px-4 py-4
- **Gaps**: 1.5-2 units between elements
- **Margins**: 0.5 units for secondary text

### Borders & Corners
- **Width**: 2px for all borders
- **Radius**: rounded-lg for badges/buttons
- **Colors**: Contextual (blue, purple, amber, etc.)
- **Transparent**: Default state for action buttons

### Shadows
- **sm**: Default for badges
- **md**: Hover state for badges and buttons
- **lg**: Not used (reserved for modals)

### Transitions
- **Duration**: 200ms for all interactions
- **Properties**: 
  - Colors
  - Transforms (scale)
  - Shadows
  - Borders

### Hover Effects
- **Scale**: 105% for badges, 110% for action buttons
- **Colors**: Darker shades or accent colors
- **Shadows**: Increase from sm to md
- **Borders**: Appear or change color
